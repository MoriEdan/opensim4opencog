It is widely acknowledged that an AGI requires a rich environment with which to interact. However, real robots suffer from a number of infelicities, from sticky servos to lack of reproducibility. A virtual robot operating in a virtual world is more controllable. At the same time, the environment must be large, complex, and unpredictable. Fortunately, virtual worlds provide such an environment. Second Life, with its large size, user created content, and lack of a clear, unidirectional goal, serves well.

Second, we expect Opensim4OpenCog to serve a wide variety of users outside the immediate AGI community, as a robust, flexible, intelligent tool for constructing automatic 'bots' in virtual worlds.
